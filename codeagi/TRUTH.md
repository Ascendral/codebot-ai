# TRUTH.md

## What CodeAGI Is

CodeAGI is an experimental autonomous cognition system.

It is a research platform for building and evaluating:
- persistent memory
- world models
- long-horizon goal pursuit
- planning and replanning
- self-critique
- verification
- learning from outcomes
- transfer across tasks

The aim is to move beyond stateless prompt-response behavior toward a system with continuity, internal state, and adaptive control.

## What CodeAGI Is Not

CodeAGI is not:
- AGI
- consciousness
- sentience
- human-level intelligence
- a solved general reasoning system
- proof that autonomous cognition has been achieved

No part of this project should imply otherwise.

## Hard Rules

1. No capability claim without measurement.
2. No intelligence claim based on fluent output alone.
3. No memory claim unless stored experience changes later behavior.
4. No learning claim unless later performance improves because of prior experience.
5. No autonomy claim unless the system sustains goal-directed behavior over time with limited intervention.
6. No reasoning claim unless it survives verification and adversarial evaluation.
7. No "AGI achieved" claim under any condition in this project's current scope.

## Real Objective

Build a system that becomes progressively less like:
- prompt in, answer out

and progressively more like:
- observe
- model
- remember
- plan
- act
- verify
- reflect
- learn
- adapt

## Required Capabilities

The project only counts as making progress if it improves in one or more of these areas:
- persistence across sessions
- world-state tracking
- long-horizon task continuity
- error detection and self-correction
- recovery after failure
- memory-guided behavior
- strategy adaptation
- transfer to unfamiliar tasks
- measurable reduction in repeated mistakes

## What Counts As Evidence

Valid evidence includes:
- benchmark improvement
- lower failure rates
- better transfer performance
- better recovery behavior
- longer autonomous task completion horizons
- improved retention and use of prior knowledge
- reproducible evaluations across runs

Invalid evidence includes:
- impressive demos without controls
- subjective feelings that it seems smarter
- longer answers
- more polished UI
- anthropomorphic behavior
- isolated anecdotal wins

## Failure Conditions

The project is failing if it becomes mostly:
- UI theater
- prompt engineering disguised as cognition
- memory storage without behavioral effect
- fake autonomy that still depends on constant rescue
- unverifiable "reasoning"
- branding-heavy and measurement-light

## Project Standard

CodeAGI should be judged by one question:

Does the system measurably become more capable, more adaptive, and more autonomous over time?

If not, we are building theater, not intelligence.
